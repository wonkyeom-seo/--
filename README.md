# 기말 자료실

`data` 폴더를 자동으로 읽어 폴더 탐색, 파일 다운로드, 모바일 PDF 보기를 제공하는 가벼운 Node.js 사이트입니다.

## 실행

Node.js 18 이상이 필요합니다.

```bash
npm install
npm start
```

기본 주소는 `http://0.0.0.0:3000`입니다. 브라우저에서는 서버의 IP 주소와 포트로 접속합니다.

개발 중 파일 변경을 감지해 서버를 다시 시작하려면:

```bash
npm run dev
```

## 환경 변수

- `PORT`: 서버 포트, 기본값 `3000`
- `HOST`: 바인딩 주소, 기본값 `0.0.0.0`
- `DATA_DIR`: 자료 폴더의 절대 또는 상대 경로, 기본값 `./data`

예시:

```bash
PORT=8080 DATA_DIR=/srv/exam-data npm start
```

자료는 서버 실행 중에도 `data` 폴더에 추가하거나 정리할 수 있습니다. 다음 탐색 또는 새로고침부터 변경된 구조가 표시됩니다.

## Ubuntu 서버

```bash
sudo apt update
sudo apt install -y nodejs npm
git clone <repository-url>
cd <repository-directory>
npm install
PORT=3000 npm start
```

UFW를 사용하는 서버에서 포트를 직접 공개하려면:

```bash
sudo ufw allow 3000/tcp
```

인터넷에 공개할 때는 Node 포트를 직접 노출하기보다 Nginx나 Caddy에서 HTTPS를 적용하고 `http://127.0.0.1:3000`으로 프록시하는 구성을 권장합니다. 프록시에서는 PDF Range 요청을 방해하지 않도록 Range 관련 헤더를 그대로 전달해야 합니다.

## PWA 및 PDF 캐시

사이트는 PWA 설치와 오프라인 모드를 지원합니다. 오프라인 모드는 기본으로 켜져 있으며 홈 화면 상단에서 끄고 켤 수 있습니다.

- 앱 화면, 뷰어, 필수 스크립트와 PDF.js 핵심 파일은 앱 셸로 캐시됩니다.
- 오프라인 모드가 켜져 있으면 열어 본 PDF가 자동 저장되고, 다음 열람부터 저장본을 먼저 사용합니다. 온라인 상태에서는 뒤에서 새 파일을 확인해 저장본을 갱신합니다.
- 폴더 목록 API는 온라인 최신 응답을 우선 사용하고, 네트워크가 끊겼을 때 마지막으로 저장된 목록으로 fallback합니다. 오프라인 검색은 저장된 PDF 파일명과 경로만 대상으로 합니다.
- 오프라인 모드를 꺼도 기존 저장 PDF는 삭제되지 않습니다. 새 자동 저장과 저장본 우선 열람만 멈추며, 저장본은 `오프라인 저장` 관리 페이지에서 열거나 삭제할 수 있습니다.
- `/content/` 직접 접근은 막지 않습니다. 잠긴 폴더는 기존 UI 흐름처럼 비밀번호를 통과한 뒤 PDF를 열 수 있고, 열린 PDF만 오프라인 저장 대상이 됩니다.

저장된 PDF는 브라우저의 사이트 데이터 삭제 또는 저장 공간 정리 시 제거될 수 있습니다.

PWA 설치와 서비스 워커는 HTTPS 환경 또는 `localhost`에서만 동작합니다. 서버 IP로 접속하는 `http://192.168.x.x:3000` 주소에서는 브라우저 보안 정책상 설치와 캐시가 활성화되지 않으므로, Ubuntu 서버에서는 HTTPS 리버스 프록시를 사용해야 합니다.
